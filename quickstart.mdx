## 🚀 Introdução

Este guia passo a passo mostra como chamar o endpoint **POST /completions**, interpretar a resposta e integrá-la ao seu aplicativo.

<Tip>

## 📂 1. Fazendo sua Primeira Chamada à API

O endpoint [**POST /completions**](/api-reference/completions/post-completions) gera um texto a partir de um prompt. Veja abaixo um exemplo de requisição:

```bash
curl --request POST \
  --url https://api.conteudo.rdstationmentoria.com.br/rest/completions \
  --header "Authorization: Bearer SUA_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "gpt-3.5-turbo",
    "prompt": "Descreva o planeta Júpiter em 2 frases.",
    "max_tokens": 60,
    "temperature": 0.5
  }'
```

<Tip>
  Se ocorrer um erro, verifique se sua chave está correta e se o endpoint está acessível.
</Tip>


## ✅ 2. Exemplo de Resposta

Uma resposta típica (HTTP 200) pode ser:

```json
{
  "completionId": "cpt-abc123",
  "output": {
    "role": "assistant",
    "content": "Júpiter é o maior planeta do sistema solar, conhecido por sua grande mancha vermelha, uma tempestade gigantesca. Ele possui um anel fino e dezenas de luas, incluindo Io e Europa.",
    "toolCall": {}
  },
  "userId": "user-abc123",
  "workspaceId": "wpc-xyz123",
  "createdAt": "2025-07-01T10:05:00.000Z",
  "updatedAt": "2025-07-01T10:05:00.000Z"
}
```

## 🛠️ 3. Usando `tool_call` com uma função personalizada

Você também pode instruir o modelo a **invocar ferramentas (functions)** de forma estruturada usando o campo `toolChoice` e fornecendo uma lista de `tools`. Isso é útil quando você quer que o modelo ative uma funcionalidade específica, como buscar informações ou executar uma ação externa com base na entrada do usuário.

Veja um exemplo:

```bash
curl --request POST \
  --url https://api.conteudo.rdstationmentoria.com.br/rest/completions \
  --header "Authorization: Bearer SUA_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "messages": [
      {
        "role": "user",
        "content": "Quero saber a previsão do tempo para amanhã em São Paulo."
      }
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "buscar_previsao_tempo",
          "description": "Busca a previsão do tempo atual para uma cidade",
          "parameters": {
            "type": "object",
            "properties": {
              "cidade": {
                "type": "string",
                "description": "Nome da cidade (ex: São Paulo)"
              },
              "data": {
                "type": "string",
                "description": "Data desejada no formato YYYY-MM-DD"
              }
            },
            "required": ["cidade", "data"]
          }
        }
      }
    ],
    "temperature": 0.7,
    "max_tokens": 100
  }'
  ```
<Tip> 
  Você pode usar essa abordagem para conectar o modelo com ações específicas, como buscar dados externos, calcular valores ou executar comandos customizados. 
</Tip>

## 🧾 Exemplo de Resposta com `toolCall`

Quando o modelo decide usar a ferramenta, a resposta incluirá um campo `toolCall` com os argumentos preenchidos:

```json
{
  "completionId": "cpt-func789",
  "output": {
    "role": "assistant",
    "toolCall": {
      "name": "buscar_previsao_tempo",
      "arguments": {
        "cidade": "São Paulo",
        "data": "2025-07-03"
      }
    }
  }
}
```

---

## 🧠 O que fazer com isso?

Agora que você tem os argumentos extraídos, seu backend pode executar a função real (`buscar_previsao_tempo`) com os parâmetros fornecidos. Esse fluxo permite que o modelo aja como interface natural para suas funcionalidades internas.

<Tip> 
  Teste este exemplo substituindo `buscar_previsao_tempo` por funções específicas do seu domínio — como agendar uma reunião, consultar um lead ou buscar dados em tempo real. 
</Tip>
